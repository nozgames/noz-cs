# Noz Game Engine

A 2d game engine written in C#

## Requirements

- .NET 10 SDK

```
winget install Microsoft.DotNet.SDK.10
```

## Project Structure

- `src/` - Noz engine (class library)
- `editor/` - Noz editor (windowed application)

## Building

Build the editor:
```
dotnet build editor/editor.csproj
```

Build the engine only:
```
dotnet build src/noz.csproj
```

## Running

```
dotnet run --project editor/editor.csproj
```

## Creating a New Game Project

### Quick Start

```bash
# 1. Create and initialize a new game project
noz --init MyGame --project ./MyGame

# 2. Navigate to the project directory
cd MyGame

# 3. Initialize git repository
git init

# 4. Add the NOZ engine as a submodule
git submodule add https://github.com/nozgames/noz-cs noz

# 5. Initialize and update the submodule
git submodule update --init --recursive

# 6. Restore NuGet packages
dotnet restore

# 7. Build the project
dotnet build

# 8. Run the desktop version
dotnet run --project platform/desktop

# 9. Open in NOZ editor
noz --project .
```

### What Gets Created

The `--init` command generates:
- Complete Visual Studio solution (`.sln`)
- Game project (class library)
- Desktop platform project (Windows executable)
- Web platform project (Blazor WebAssembly)
- Template game code (`Game.cs`, `GameConfig.cs`)
- All necessary configuration files
- Git configuration (`.gitignore`, `.gitmodules`)
- Empty asset library structure

```
│   │   └── Program.cs
│   └── web/                # Web platform (Blazor)
│       ├── MyGame.Web.csproj
│       ├── Program.cs

│       └── wwwroot/
├── library/                # Compiled assets (generated by editor)
├── assets/                 # Source assets
├── noz/                    # NOZ engine (git submodule)
└── editor.cfg              # Editor configuration
```
